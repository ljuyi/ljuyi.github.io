vue重构在线考试系统

Posted on 2017-06-16

前言

学习了vue一段时间了，在网上跟着老师用vue做了一遍饿了么的几个页面，觉得学习一个新
技术还是项目驱动学起来更加的快，于是把之前和项目小组的小伙伴们写的在线考试系统重新用vue做一遍，之前做的时候只想着实现功能，导致页面性能根本没有考虑，每次点击跳转都慢的。。。虽然这个考试系统没有投入使用，但把它做好做优秀也会很有成就感，说不定将来就用上了呢。项目地址

介绍

先来介绍一下这个项目吧，主要实现的功能是在线考试系统的后台页面开发，管理员通过入口网页登陆进入到管理页面，可以查看课程信息，题目
信息，人员信息，考试信息，根据echarts图标总结考生成绩情况，这是总的功能划分，其次每个信息列表还有信息筛选的功能，可以指定查看特定一类的人员信息，课程或者题目信息
为了解决页面多而繁琐，我使用了vue-router做前端路由，将公用的部分组件化，实现代码重用，相比于原来每个页面都有一个tab来说，性能上
也能有所优化。vuex用来存储登陆用户的信息，可以使登陆后的管理页面获取到登陆页面用户输入信息。
静态资源我没有使用json数据进行存储在项目中，而是托管在七牛云上，之前没有使用过，想借此机会学习一下，在项目中我在api目录下实现了
向七牛云上传更新后的文件，使用express搭建的服务器，关于七牛云的api，可以google搜索七牛云 SDK，我在这里就不贴链接了。（这里要说一下七牛云的静态资源可能不是实时更新的，因为我成功上传更新了的文件后，登陆上去看发现时间是更新了的，但是内容并没有，而我在测试时打印出来上传的文件是更新了的）

解决问题

最近遇到一个问题觉得挺值得记录一下的：
因为项目中有一个功能是点击题目可以在弹出框中显示题目的信息，我需要获得用户点击的tr标签序号，然后将序号和整个题目数组派发给alert组件，告诉alert组件应该显示哪个题目的信息。在我点击的时候浏览器console面板显示在alert组件内没有题目数组的某个属性值，其实在看到这个问题时就应该想到是渲染时间的问题，一定是我在页面渲染时还没有获取到这个数组，所以也就找不到相关属性，于是我在调用这个组件的vue文件中发现，我在定义alert组件的显示与不显示时用的是v-show,熟悉vue的人应该都知道，v-show和v-if的区别，前者是在渲染树中生成了该节点，改变的不过是display的属性值，但后者是从渲染树中插入和删除，所以可想而知，我在调用父组件异步获取资源的同时，这个alert组件就已经被插入到渲染树中了，但是当我还没有获取到资源信息并传递给该组件时，alert组件是拿不到这个题目数组的，所以就出现这个属性的值undefined的情况了。
