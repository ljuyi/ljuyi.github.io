知识点

在http1.0版本里，通信是无状态的，就是说，客户端想要给服务器端发送消息，就只能建立连接-》发送消息-》连接断开，如果想重新发送，就必须又要重新建立新的连接，十分麻烦，这很明显增加了客户端和服务端的时间成本。
在websocket之前还有两种方法（具体出现时间不太清楚）,ajax轮询，长轮询。
先来说说ajax轮询：

看起来和普通的ajax请求似乎并无差别，唯一不同的是客户端每隔一段时间就会发送请求，其实就是用setInterval将一个ajax请求包起来，自己设定请求间隔时间
长轮询:

长轮询比ajax轮询上了一个档次，客户端有请求时向服务端发送请求，但是服务端可以不立即进行回复，而是等到有消息了再进行回复，而这段时间，客户端和服务端是一直保持连接状态的，当服务端有消息回复时，这种连接状态就断开了，客户端需要再次像服务端建立连接请求，comet就技术使用了这种方法，不过我没有用过，不太了解

终于到了websocket
上面的两种方法都没有解决服务器端不能向客户端主动发送请求的问题，这样的弊端就是如果有hin多hin多的客户端同时发送请求，服务器端就会忙不过来，于是。。503。。websocket就完美地解决了这个问题，客户端只需要向服务器端发送一次建立连接的请求，这种连接状态就会一直保持，直到客户端关闭请求，服务器端也可以主动向服务器端进行消息的发送，是不是很棒~

怎样分房间聊天？
因为socket.io的广播方式是对全体客户端进行广播，要是想实现分房间广播就有两种方式，
一种是在服务端确定广播的对象，一种是在客户端。要是在服务器端确定的话，就要绑定许多许多事件，每建立一个房间，就要绑定给这个房间绑定一遍所有的事件，然后根据事件的名称进行对象确定，这无疑增加了服务器端的资源耗费，于是我就采用了在客户端进行筛选广播对象，只用在客户端广播消息时增加一个房间号信息，在客户端进行房间号匹配，如果相等则将信息更新。但是我觉得这种方法也有弊端，别的房间的信息有可能会被窃取。有更好的方法再更新。

js中的基本类型和引用类型

Posted on 2016-11-16

js数据类型

js中的数据类型有以下几种：

number boolean undefined object function string null

js中的数据类型主要有两种：** 基本类型 ** 和 ** 引用类型 **

####基本类型
number boolean string undefined null

基本类型存储在栈内存中,基本类型的复制会在栈内存中重新开辟一个空间存储这个新的变量，也就是说，改变其中一个变量的值，另一个值不会改变

引用类型存储在堆内存中，但是在栈内存中会开辟一个空间存储这个引用类型在堆内存的地址，相当于在栈内存中存了一个指针指向堆内存中的该引用类型
如果发生引用类型的复制，则相当于在栈内存中又开辟一块空间存储一个指针，指向相同的堆内存中的引用类型，也就是说，** 修改其中一个引用类型的值，则另一个的值也会改变 **

需要注意的是，基本类型和引用类型还有一个区别：** 基本类型是没有属性和方法的，只有引用类型有 **

基本类型如果定义了属性或者方法，则会立即销毁，所以执行下一句的时候，string的index属性其实已经不存在了
但是为什么String 类型还有那么多方法呢？

这是因为在执行这段代码时，在内部进行了如下操作：

(1)创建一个String类型的实例(使用 new 操作符创建的引用类型的实例，在执行流离开当前作用域之前都一直保存在内存中。而自动创建的基本包装类型的对象，则只存在于一
行代码的执行瞬间，然后立即被销毁 )

(2)在实例上调用指定方法

(3)销毁该实例

所以，虽然定义的是一个基本类型String，但是在调用String的方法时定义了一个 基本包装类型 的String
除过String属于 基本包装类型 ，还有boolean number也属于 基本包装类型
